# Travail de 24157 (Réservations & File d'attente)

**Statut : ✅ Terminé**

Implémentation de la gestion des réservations avec file d'attente FIFO, statuts PENDING/READY/CANCELLED/COMPLETED, et dates de suivi.

---

## 1. ENUM

**Chemin : `src/main/java/supnum/projet/Library/data/entities/enums/ReservationStatus.java`**
```java
public enum ReservationStatus {
    PENDING, READY, CANCELLED, COMPLETED
}
```

---

## 2. ENTITÉ JPA

### Reservation
- `id` (auto-généré)
- `member` → `@ManyToOne` (LAZY, `@NotFound(IGNORE)`)
- `book` → `@ManyToOne` (LAZY, `@NotFound(IGNORE)`)
- `queuePosition` → position dans la file (FIFO)
- `reservationDate` → `LocalDateTime`, date de réservation
- `expirationDate` → `LocalDateTime`, date d'expiration (optionnelle)
- `status` → `ReservationStatus` (PENDING par défaut)
- Lombok : `@Builder`, `@Getter`, `@Setter`, `@NoArgsConstructor`, `@AllArgsConstructor`

---

## 3. REPOSITORY

**Chemin : `src/main/java/supnum/projet/Library/data/repositories/ReservationRepository.java`**
- `findByBookAndStatusOrderByQueuePositionAsc(Book, ReservationStatus)` → `List<Reservation>` (file d'attente triée)
- `findMaxQueuePositionForBook(Book)` → `Integer` (MAX queue_position pour un livre, via `@Query`)
- `findByMemberAndBookAndStatus(Member, Book, ReservationStatus)` → `List<Reservation>` (vérification réservation existante)

---

## 4. DTOs

### Requête
**ReservationDTO** : `memberId` (@NotNull), `bookId` (@NotNull)

### Réponse
**ReservationResponse** : `id`, `memberId`, `memberEmail`, `bookId`, `bookTitle`, `queuePosition`, `reservationDate`, `expirationDate`, `status`

---

## 5. SERVICE

**Chemin : `src/main/java/supnum/projet/Library/services/ReservationService.java`**

| Méthode | Description |
|---------|-------------|
| `findAll(Pageable)` | Liste paginée des réservations |
| `findById(Long)` | Détail d'une réservation |
| `reserve(ReservationDTO)` | Créer une réservation (calcule `queuePosition` = MAX+1, set `reservationDate`) |
| `cancel(Long)` | Annuler une réservation PENDING → CANCELLED |
| `getQueueForBook(Long)` | File d'attente pour un livre (ordonnée par position) |

Règles métier :
- La position dans la file est calculée automatiquement (`MAX(queuePosition) + 1`)
- Une réservation PENDING ne peut être annulée que si elle est en attente
- `BusinessException` levée pour les violations métier
- `ResourceNotFoundException` pour les entités introuvables
