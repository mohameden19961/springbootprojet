# Travail de 24139 (Tables de référence : Langue, Nationalité, Catégorie, Éditeur, Auteur)

**Statut : ✅ Terminé**

Implémentation des tables de référence (Language, Nationality, Category, Publisher, Author) avec CRUD complet, pagination, soft-delete et seed initial via `SeedController`.

---

## 1. ENTITÉS JPA (toutes étendent `BaseEntity` pour soft-delete, Lombok `@Builder @Getter @Setter @NoArgsConstructor @AllArgsConstructor`)

### Language
- `code` (String, **PK**) — ex: `fr`, `en`
- `name` (String) — ex: `Français`, `English`

### Nationality
- `code` (String, **PK**) — ex: `fr`, `en`
- `name` (String) — ex: `Française`, `Anglaise`

### Category (`@SQLRestriction("deleted = false")`)
- `id` (auto-généré)
- `name` (String) — ex: `Science-Fiction`, `Roman`

### Publisher (`@SQLRestriction("deleted = false")`)
- `id` (auto-généré)
- `name` (String) — ex: `Gallimard`

### Author (`@SQLRestriction("deleted = false")`)
- `id` (auto-généré)
- `name` (String) — ex: `Victor Hugo`
- `nationality` → `@ManyToOne` (LAZY, `@NotFound(IGNORE)`)

---

## 2. REPOSITORIES

| Repository | Méthode spécifique |
|------------|--------------------|
| `LanguageRepository` | `findById(String)` standard |
| `NationalityRepository` | `findById(String)` standard |
| `CategoryRepository` | `findByName(String)` → `Optional<Category>` |
| `PublisherRepository` | `findByName(String)` → `Optional<Publisher>` |
| `AuthorRepository` | — |

Les repository `Language` et `Nationality` n'étendent **pas** `BaseEntity` (pas de soft-delete, ce sont des données de référence stables).

---

## 3. SERVICES

Tous les services suivent le même pattern :
- `findAll(Pageable)` → `Page<XResponse>` (sauf Language/Nationality qui retournent `List`)
- `findById(id)` → `XResponse`
- `create(XDTO)` → `XResponse` (vérifie unicité du nom si applicable)
- `update(id, XDTO)` → `XResponse`
- `delete(id)` → soft-delete (sauf Language/Nationality)

### Services avec vérification d'unicité
- **CategoryService** : `findByName` avant création/mise à jour
- **PublisherService** : `findByName` avant création/mise à jour

---

## 4. DTOs

### Entités avec auto-génération ID (Category, Publisher, Author)
- **DTO** : `name` (+ `nationalityCode` pour Author)
- **Response** : `id`, `name` (+ `nationalityCode`, `nationalityName` pour Author)

### Entités avec PK String (Language, Nationality)
- **DTO** : `code`, `name`
- **Response** : `code`, `name`

---

## 5. CONTROLLERS

Tous les contrôleurs sont sous `@RequestMapping("/api/<entity>")` avec le même pattern CRUD + pagination :

| Méthode | Endpoint | Description |
|---------|----------|-------------|
| GET | `/api/<entity>` | Liste (paginée sauf Language/Nationality) |
| GET | `/api/<entity>/{id}` | Détail |
| POST | `/api/<entity>` | Création |
| PUT | `/api/<entity>/{id}` | Modification |
| DELETE | `/api/<entity>/{id}` | Soft-delete |

Contrôleurs :
- **LanguageController** — `/api/languages` (GET seulement en liste, sans pagination, lecture seule)
- **NationalityController** — `/api/nationalities` (GET seulement en liste, sans pagination, lecture seule)
- **CategoryController** — `/api/categories`
- **PublisherController** — `/api/publishers`
- **AuthorController** — `/api/authors`
