### 6. Book
```java
@Entity
@Table(name = "book")
public class Book {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false)
    private String title;           // "Clean Code"
    
    @Column(nullable = false, unique = true)
    private String isbn;            // "978-0132350884"
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "language_code", nullable = false)
    private Language language;      // Relation vers Language
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "category_id", nullable = false)
    private Category category;      // Relation vers Category
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "publisher_id", nullable = false)
    private Publisher publisher;    // Relation vers Publisher
    
    @ManyToMany(fetch = FetchType.LAZY, cascade = CascadeType.ALL)
    @JoinTable(
        name = "book_author",
        joinColumns = @JoinColumn(name = "book_id"),
        inverseJoinColumns = @JoinColumn(name = "author_id")
    )
    private Set<Author> authors;    // Relation N-N vers Author
    
    @OneToMany(mappedBy = "book", fetch = FetchType.LAZY, cascade = CascadeType.ALL)
    private Set<BookItem> bookItems;    // Exemplaires du livre
    
    @OneToMany(mappedBy = "book", fetch = FetchType.LAZY, cascade = CascadeType.ALL)
    private Set<Reservation> reservations;   // Réservations du livre
    
    @Column(nullable = false, columnDefinition = "BOOLEAN DEFAULT false")
    private Boolean deleted;        // Soft delete
}
```

**Utilité** : L'entité PRINCIPALE - livre intellectuel  
**Concept Clé** : Un Book ≠ BookItem (Book = concept, BookItem = exemplaire physique)  
**Relations** :
- N-1 vers Language (LAZY)
- N-1 vers Category (LAZY)
- N-1 vers Publisher (LAZY)
- N-N vers Author via table JOIN (LAZY)
- 1-N vers BookItem (LAZY)
- 1-N vers Reservation (LAZY)

**Soft Delete** : Oui  
**Exemple** :
```
Id=1, Title="Clean Code", ISBN="978-0132350884"
Language="en", Category=1, Publisher=1
Authors={1, 2, 3}, BookItems={1, 2, 3}
```

### 7. BookItem (CONCEPT CLÉE)
```java
@Entity
@Table(name = "book_item")
public class BookItem {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false, unique = true)
    private String barcode;        // "BOOK-001-2024" (UNIQUE)
    
    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private BookItemStatus status; // AVAILABLE, BORROWED, RESERVED...
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "book_id", nullable = false)
    private Book book;             // Relation vers Book
    
    @OneToOne(mappedBy = "bookItem", fetch = FetchType.LAZY)
    private Borrow borrow;         // Emprunt actuel
    
    @Version
    private Long version;          // OPTIMISTIC LOCKING - TRÈS IMPORTANT
    
    @Column(nullable = false, columnDefinition = "BOOLEAN DEFAULT false")
    private Boolean deleted;       // Soft delete
}
```

**CONCEPT CLÉE** : BookItem = EXEMPLAIRE PHYSIQUE  
**Exemple** : Le livre "Clean Code" existe en 3 exemplaires :
- BookItem 1 : barcode="CC-001", status=AVAILABLE
- BookItem 2 : barcode="CC-002", status=BORROWED
- BookItem 3 : barcode="CC-003", status=AVAILABLE

**Version (Optimistic Lock)** :
```
Sans version : 2 utilisateurs empruntent le même exemplaire SIMULTANÉMENT
Avec version  : Hibernate détecte le conflit et lance OptimisticLockException
```

**Relations** :
- N-1 vers Book (LAZY)
- 1-1 vers Borrow (LAZY, mapped side)

**Soft Delete** : Oui

### 9. Borrow (Table Transactionnelle)
```java
@Entity
@Table(name = "borrow")
public class Borrow {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "member_id", nullable = false)
    private Member member;         // Relation vers Member
    
    @OneToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "book_item_id", nullable = false)
    private BookItem bookItem;     // L'exemplaire emprunté
    
    @Column(nullable = false)
    private Integer renewalCount;  // Nombre de renouvellements
    
    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private BorrowStatus status;   // ACTIVE, RETURNED, OVERDUE, LOST
    
    @Column(nullable = false, updatable = false)
    private LocalDateTime borrowedAt;   // Date d'emprunt (immuable)
    
    @Column
    private LocalDateTime returnedAt;   // Date de retour
    
    @Column
    private LocalDateTime dueDate;      // Date d'échéance
    
    @Column(nullable = false, columnDefinition = "BOOLEAN DEFAULT false")
    private Boolean deleted;       // Soft delete
}
```

**Utilité** : Tracer tous les emprunts (historique complet)  
**Cycle de Vie** :
```
1. Création : status=ACTIVE, borrowedAt=now, dueDate=now+30j
2. Renouvellement : renewalCount++, dueDate=now+30j
3. Retour : status=RETURNED, returnedAt=now
4. Suppression logique : deleted=true (conserve l'historique)
```

**RenewalCount** : Max 3 renouvellements par emprunt  
**Soft Delete** : Oui

### 10. Reservation (Table Transactionnelle)
```java
@Entity
@Table(name = "reservation")
public class Reservation {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "member_id", nullable = false)
    private Member member;         // Qui réserve ?
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "book_id", nullable = false)
    private Book book;             // Quel livre ?
    
    @Column(nullable = false)
    private Integer queuePosition; // Position en file d'attente
    
    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private ReservationStatus status; // PENDING, READY, CANCELLED, COMPLETED
    
    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;  // Date de réservation (immuable)
    
    @Column(nullable = false, columnDefinition = "BOOLEAN DEFAULT false")
    private Boolean deleted;       // Soft delete
}
```

**Utilité** : Gestion de la file d'attente quand aucun exemplaire dispo  
**Cycle de Vie** :
```
1. Création : status=PENDING, queuePosition=auto
2. Livre disponible : status=READY (notification)
3. Emprunt complété : status=COMPLETED
4. Annulation : status=CANCELLED + réorganisation queue
```

**QueuePosition** : Calculée automatiquement (1, 2, 3, ...)  
**Soft Delete** : Oui

### 11. BookAuthor (Table de Jointure)
```java
@Entity
@Table(name = "book_author")
public class BookAuthor {
    @EmbeddedId
    private BookAuthorId id;       // Clé composite
    
    @ManyToOne(fetch = FetchType.LAZY)
    @MapsId("bookId")
    @JoinColumn(name = "book_id")
    private Book book;             // Relation vers Book
    
    @ManyToOne(fetch = FetchType.LAZY)
    @MapsId("authorId")
    @JoinColumn(name = "author_id")
    private Author author;         // Relation vers Author
    
    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private AuthorRole role;       // MAIN_AUTHOR, CO_AUTHOR, EDITOR
}
```

**Utilité** : Gérer la relation N-N Book-Author avec rôle  
**Pourquoi ?** : Un livre peut avoir plusieurs auteurs et rôles différents  
**Exemple** : Le livre "Clean Code" peut avoir :
- Robert Martin (MAIN_AUTHOR)
- Co-author1 (CO_AUTHOR)
- Editor1 (EDITOR)


## 🔒 Optimistic Locking

Implémenté UNIQUEMENT sur BookItem :
```java
@Version
private Long version;
```

**Pourquoi BookItem ?**
- C'est l'entité critique pour les accès concurrents
- Deux membres pourraient emprunter le MÊME exemplaire simultanément
- Hibernate détecte et lance OptimisticLockingFailureException

## 🚀 Points Clés à Retenir

1. **Book vs BookItem**
   - Book = concept intellectuel (1 titre)
   - BookItem = exemplaire physique (N copies)

2. **Soft Delete**
   - Toutes les suppressions sont logiques
   - La colonne `deleted` marque les suppressions

3. **Optimistic Lock**
   - La colonne `version` prévient les conflits concurrents
   - Critique pour BookItem

4. **Relations LAZY**
   - Chargement à la demande
   - Meilleure performance

5. **DTOs pour Validation**
   - Tous les DTOs utilisent Jakarta Validation
   - Validation automatique par Spring
