# Travail de 24014 (Membres & Emprunts)

**Statut : ✅ Terminé**

Implémentation des entités `Member` et `Borrow`, des énums associés, et de la logique métier complète : emprunt, retour, renouvellement avec limites, annulation automatique des réservations, et suivi temporel.

---

```java
public enum MemberType {
    STUDENT, TEACHER, EXTERNAL
}
```

**Chemin : `src/main/java/supnum/projet/Library/data/entities/enums/BorrowStatus.java`**
```java
public enum BorrowStatus {
    ACTIVE, RETURNED, OVERDUE, LOST
}
```

## 2. ENTITÉS JPA

### Member (`@SQLRestriction("deleted = false")`, extends BaseEntity)
- `id` (auto-généré), `email` (unique), `memberType`, `maxBorrows`
- Lombok : `@Builder`, `@Getter`, `@Setter`, `@NoArgsConstructor`, `@AllArgsConstructor`

### Borrow
- `id` (auto-généré)
- `member` → `@ManyToOne` (LAZY, `@NotFound(IGNORE)`)
- `bookItem` → `@ManyToOne` (LAZY, `@NotFound(IGNORE)`)
- `renewalCount` → compteur de renouvellements (défaut 0)
- `borrowDate` → `LocalDateTime`, date d'emprunt
- `dueDate` → `LocalDate`, date de retour prévue (J+14)
- `returnDate` → `LocalDateTime`, date de retour réelle
- `status` → `BorrowStatus` (ACTIVE par défaut)
- Lombok : `@Builder`, `@Getter`, `@Setter`, `@NoArgsConstructor`, `@AllArgsConstructor`

## 5. SERVICES

### MemberService
- `findAll(Pageable)` → `Page<MemberResponse>`
- `findById(Long)` → `MemberResponse`
- `create(MemberDTO)` → `MemberResponse` (vérifie email unique)
- `update(Long, MemberDTO)` → `MemberResponse`
- `delete(Long)` → soft-delete

### BorrowService
- `findAll(Pageable)` → `Page<BorrowResponse>`
- `findById(Long)` → `BorrowResponse`
- `borrowBook(Long memberId, String barcode)` → `BorrowResponse`
  - Vérifie que l'exemplaire est AVAILABLE
  - Vérifie la limite d'emprunts du membre (`maxBorrows`)
  - Passe l'exemplaire en BORROWED
  - Set `borrowDate = now`, `dueDate = J+14`
  - Annule les réservations PENDING du membre pour ce livre
- `returnBook(Long borrowId)` → `BorrowResponse`
  - Passe le statut en RETURNED
  - Set `returnDate = now`
  - Repasse l'exemplaire en AVAILABLE
- `renewBorrow(Long borrowId)` → `BorrowResponse` (max 3 renouvellements)

Exceptions métier :
- `ResourceNotFoundException` (404) pour entité introuvable
- `BusinessException` (400) pour violations des règles (limite atteinte, exemplaire non disponible, etc.)
