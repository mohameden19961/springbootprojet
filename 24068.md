## Étape 1 : Classe de base pour le Soft Delete
Tes collègues auront besoin du concept de "Soft Delete". Crée une classe abstraite dont les autres entités hériteront.

**Fichier : `src/main/java/com/library/domain/BaseEntity.java`**
```java
package com.library.domain;

import jakarta.persistence.MappedSuperclass;
import jakarta.persistence.Column;

@MappedSuperclass
public abstract class BaseEntity {
    
    @Column(name = "deleted", nullable = false)
    private boolean deleted = false;

    public boolean isDeleted() { return deleted; }
    public void setDeleted(boolean deleted) { this.deleted = deleted; }
}
```

## Étape 2 : Configuration de Spring Security
Tu dois protéger les API. Voici une configuration basique (Basic Auth ou prête pour JWT) que tu vas intégrer.

**Fichier : `src/main/java/com/library/security/SecurityConfig.java`**
```java
package com.library.security;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable()) // Désactivé pour une API REST classique ou JWT
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll() // Routes publiques
                .requestMatchers("/api/admin/**").hasRole("ADMIN") // Seulement Admin
                .anyRequest().authenticated()
            )
            .httpBasic(basic -> {}); // Utilisation de Basic Auth pour commencer

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

## Étape 3 : Gestionnaire global des Exceptions
C'est crucial pour renvoyer des erreurs propres aux clients (400, 404, 500).

**Fichier : `src/main/java/com/library/exception/GlobalExceptionHandler.java`**
```java
package com.library.exception;

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import java.util.HashMap;
import java.util.Map;

@RestControllerAdvice
public class GlobalExceptionHandler {

    // Gestion des ressources non trouvées
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<Map<String, String>> handleNotFound(ResourceNotFoundException ex) {
        Map<String, String> response = new HashMap<>();
        response.put("error", ex.getMessage());
        return new ResponseEntity<>(response, HttpStatus.NOT_FOUND);
    }

    // Gestion des erreurs de validation (DTOs)
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String, String>> handleValidationExceptions(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors().forEach(error -> 
            errors.put(error.getField(), error.getDefaultMessage()));
        return new ResponseEntity<>(errors, HttpStatus.BAD_REQUEST);
    }
}
```

*Note : N'oublie pas de créer la classe `ResourceNotFoundException` qui hérite de `RuntimeException`.*
