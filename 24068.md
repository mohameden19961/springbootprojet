## 🏗️ Architecture Implémentée

### Approche Architecturale

Le projet suit une **architecture en couches** professionnelle :

```
┌─────────────────────────────────────┐
│      REST Controllers Layer          │ ← BookController, MemberController, etc.
├─────────────────────────────────────┤
│      Services Layer (Logique Métier) │ ← BookService, BorrowService, etc.
├─────────────────────────────────────┤
│      Repositories Layer (Data Access)│ ← BookRepository, MemberRepository, etc.
├─────────────────────────────────────┤
│      Entities Layer (Modèle)         │ ← Book, Member, BookItem, etc.
├─────────────────────────────────────┤
│      Database Layer                  │ ← MySQL
└─────────────────────────────────────┘
```

### Principes de Conception

1. **Séparation des Responsabilités**
   - Controllers : Gèrent les requêtes HTTP
   - Services : Implémentent la logique métier
   - Repositories : Accès aux données
   - DTOs : Transfert de données sans exposer les entités

2. **DRY (Don't Repeat Yourself)**
   - Lombok pour réduire le boilerplate
   - Mappers centralisés
   - Héritage et composition réutilisables

3. **SOLID Principles**
   - Single Responsibility : Chaque classe a une responsabilité unique
   - Open/Closed : Extensible sans modifier
   - Liskov Substitution : Interfaces cohérentes
   - Interface Segregation : Interfaces spécifiques
   - Dependency Inversion : Injection de dépendances

#### Paramètres Clés Expliqués

| Paramètre | Valeur | Explication |
|-----------|--------|-------------|
| `spring.datasource.url` | `jdbc:mysql://localhost:3306/library_db` | Connexion MySQL locale |
| `spring.datasource.username` | `supnum` | Utilisateur MySQL |
| `spring.datasource.password` | `Supnum` | Mot de passe MySQL |

### Structure des Tables (Auto-générée par Hibernate)

Hibernate crée automatiquement les tables selon les entités. Voici la structure créée :

```sql
-- Tables de référence
CREATE TABLE language (
    code VARCHAR(10) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE
);

CREATE TABLE nationality (
    code VARCHAR(10) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE
);

-- Tables principales
CREATE TABLE category (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    deleted BOOLEAN DEFAULT FALSE
);

CREATE TABLE publisher (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    deleted BOOLEAN DEFAULT FALSE
);

CREATE TABLE author (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    nationality_code VARCHAR(10),
    deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (nationality_code) REFERENCES nationality(code)
);

CREATE TABLE book (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    isbn VARCHAR(20) NOT NULL UNIQUE,
    language_code VARCHAR(10) NOT NULL,
    category_id BIGINT NOT NULL,
    publisher_id BIGINT NOT NULL,
    deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (language_code) REFERENCES language(code),
    FOREIGN KEY (category_id) REFERENCES category(id),
    FOREIGN KEY (publisher_id) REFERENCES publisher(id)
);

CREATE TABLE book_item (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    barcode VARCHAR(50) NOT NULL UNIQUE,
    status ENUM('AVAILABLE', 'BORROWED', 'RESERVED', 'LOST', 'DAMAGED') DEFAULT 'AVAILABLE',
    book_id BIGINT NOT NULL,
    version BIGINT DEFAULT 0,
    deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (book_id) REFERENCES book(id)
);

CREATE TABLE member (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    member_type ENUM('STUDENT', 'TEACHER', 'EXTERNAL') NOT NULL,
    max_borrows INT NOT NULL,
    deleted BOOLEAN DEFAULT FALSE
);

CREATE TABLE borrow (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    member_id BIGINT NOT NULL,
    book_item_id BIGINT NOT NULL,
    renewal_count INT DEFAULT 0,
    status ENUM('ACTIVE', 'RETURNED', 'OVERDUE', 'LOST') DEFAULT 'ACTIVE',
    borrowed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    returned_at DATETIME,
    due_date DATETIME,
    deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (member_id) REFERENCES member(id),
    UNIQUE KEY (book_item_id),
    FOREIGN KEY (book_item_id) REFERENCES book_item(id)
);

CREATE TABLE reservation (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    member_id BIGINT NOT NULL,
    book_id BIGINT NOT NULL,
    queue_position INT NOT NULL,
    status ENUM('PENDING', 'READY', 'CANCELLED', 'COMPLETED') DEFAULT 'PENDING',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (member_id) REFERENCES member(id),
    FOREIGN KEY (book_id) REFERENCES book(id)
);

CREATE TABLE book_author (
    book_id BIGINT NOT NULL,
    author_id BIGINT NOT NULL,
    role ENUM('MAIN_AUTHOR', 'CO_AUTHOR', 'EDITOR') NOT NULL,
    PRIMARY KEY (book_id, author_id),
    FOREIGN KEY (book_id) REFERENCES book(id),
    FOREIGN KEY (author_id) REFERENCES author(id)
);
```

## 🔍 Points de Configuration Critiques

### 1. Connexion MySQL

**Problème** : Erreur "Access Denied"
**Solution** : 
```bash
# Vérifier MySQL en cours d'exécution
mysql -u supnum -pSupnum -h localhost

# Créer l'utilisateur si manquant
CREATE USER 'supnum'@'localhost' IDENTIFIED BY 'Supnum';
GRANT ALL PRIVILEGES ON library_db.* TO 'supnum'@'localhost';
FLUSH PRIVILEGES;
```

### 2. Port Already in Use

**Problème** : Port 8080 déjà utilisé
**Solution** : Dans application.properties
```properties
server.port=8081  # ou autre port disponible
```

### 3. Hibernate DDL

**Problème** : Tables ne se créent pas
**Solution** : Vérifier dans application.properties
```properties
spring.jpa.hibernate.ddl-auto=update  # À la place de "none"
```

## 📊 Performance et Optimisations

### Optimisations Implémentées

1. **Batch Processing**
   ```properties
   hibernate.jdbc.batch_size=20
   hibernate.order_inserts=true
   hibernate.order_updates=true
   ```

2. **Query Optimization**
   - Lazy loading sur les associations N-N et 1-N
   - Join Fetch pour éviter N+1 queries quand nécessaire

3. **Connection Pooling**
   - Spring utilise HikariCP par défaut
   - Configuration automatique

4. **Caching**
   - À implémenter avec Spring Cache si besoin
