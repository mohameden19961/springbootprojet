## 3. Sécurité JWT (Spring Security)

### 3.1 SecurityConfig
**Chemin : `src/main/java/supnum/projet/Library/security/SecurityConfig.java`**
- CSRF désactivé (API stateless)
- Sessions STATELESS (JWT)
- Route publique : `POST /api/auth/login`
- Routes ADMIN : `/api/admin/**` (hasRole("ADMIN"))
- Toute autre route : authentifiée
- Filtre JWT ajouté avant `UsernamePasswordAuthenticationFilter`
- Bean `PasswordEncoder` (BCrypt)

### 3.2 JwtUtil
**Chemin : `src/main/java/supnum/projet/Library/security/JwtUtil.java`**
- Génération token JWT avec HMAC-SHA256 (jjwt 0.12.6)
- Claims : `subject` (username), `role`, `iat`, `exp`
- Propriétés injectées : `jwt.secret`, `jwt.expiration` (défaut 24h)
- Méthodes : `generateToken`, `getUsernameFromToken`, `getRoleFromToken`, `validateToken`

### 3.3 JwtAuthenticationFilter
**Chemin : `src/main/java/supnum/projet/Library/security/JwtAuthenticationFilter.java`**
- Étend `OncePerRequestFilter`
- Extrait le token Bearer du header `Authorization`
- Valide le token et charge l'utilisateur via `UserDetailsService`
- Positionne l'authentification dans `SecurityContextHolder`

### 3.4 UserDetailsServiceImpl
**Chemin : `src/main/java/supnum/projet/Library/security/UserDetailsServiceImpl.java`**
- Charge l'utilisateur depuis `UserRepository`
- Retourne un `UserDetails` avec le rôle prefixé `ROLE_`
